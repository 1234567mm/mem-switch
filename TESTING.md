# Mem-Switch 用户测试指南

## 测试目标

验证 Phase 2 & Phase 3 新功能：
1. 批量导入与后台处理
2. 记忆管理增强
3. 统一搜索中心
4. 首次使用引导 (Onboarding)

## 测试环境

- **前端**: http://localhost:5173
- **后端**: http://localhost:8000
- **版本**: v0.1.0

## 测试场景

### 场景 1: 首次启动体验 (新用户)

**步骤**:
1. 清除浏览器缓存或打开隐私模式
2. 访问 http://localhost:5173
3. 完成 Onboarding 向导 (5 步)

**预期结果**:
- [ ] 显示欢迎界面
- [ ] Ollama 配置步骤可测试连接
- [ ] 模型选择步骤显示推荐模型
- [ ] 可跳过或执行数据导入
- [ ] 完成后进入主界面

### 场景 2: 批量导入对话

**准备**: 准备 3-5 个对话文件（JSON/Markdown 格式）

**步骤**:
1. 点击侧边栏「对话导入」
2. 选择数据源平台（如 Claude Code）
3. 拖拽或选择多个文件
4. 点击「开始导入」
5. 观察进度显示

**预期结果**:
- [ ] 可选择多个文件
- [ ] 显示文件列表
- [ ] 进度条显示正确（成功/跳过/失败）
- [ ] 已导入的会话自动跳过
- [ ] 完成后显示统计

**记录问题**: _______________

---

### 场景 3: 记忆管理

**步骤**:
1. 点击侧边栏「记忆库」
2. 找到任意记忆
3. 点击「编辑」按钮
4. 修改内容并保存
5. 点击「失效」按钮
6. 查看统计信息（调用次数）

**预期结果**:
- [ ] 编辑弹窗正常显示
- [ ] 保存后内容更新
- [ ] 失效后显示「已失效」标签
- [ ] 恢复功能正常
- [ ] 调用次数显示正确

**记录问题**: _______________

---

### 场景 4: 统一搜索

**步骤**:
1. 点击侧边栏顶部搜索按钮 🔍
2. 勾选「记忆库」和「知识库」
3. 输入搜索词
4. 查看搜索结果
5. 再次搜索相同内容（测试缓存）

**预期结果**:
- [ ] 搜索面板从侧边栏滑出
- [ ] 可选择搜索范围
- [ ] 返回搜索结果
- [ ] 第二次搜索显示「缓存」标识
- [ ] 显示搜索历史

**记录问题**: _______________

---

### 场景 5: Toast 通知

**步骤**:
1. 执行成功操作（如保存设置）
2. 执行失败操作（如搜索不存在的记忆）
3. 观察通知显示

**预期结果**:
- [ ] 成功通知显示绿色 ✓
- [ ] 错误通知显示红色 ✕
- [ ] 通知自动消失（4-6 秒）
- [ ] 可手动关闭通知
